What is a ghost font?

A ghost font writes a message using motion. The text is a field of random dots that drift in one direction, sitting inside a background of dots that drift in another. Human vision separates the two motions instantly, so the message is legible while it plays. Pause it and the letters disappear — no single frame holds any difference between text and background, so screenshots, frame extraction, and OCR all come back empty.

Because AI agents read video frame by frame, the pitch is simple: this is the text AI cannot read. It is sold as an anti-AI watermark — content that agents supposedly cannot summarize, scrape, or act on.

How the decoder breaks it

The claim has one flaw: the message is the motion. So the decoder reads the motion, not the frames. Four steps, no model weights, no GPU.

STEP 1

Dense optical flow

A motion vector for every pixel between consecutive frames (OpenCV DIS / Farnebäck).

STEP 2

Subtract the background

The median flow is the background motion. Pixels moving against it are candidate text.

STEP 3

Register the drift

Phase correlation aligns each frame so drifting glyphs stack sharply instead of smearing.

STEP 4

Accumulate & read

Scores sum over hundreds of pairs, noise cancels, letters remain — then OCR reads them.

The security claim, tested

They claim it's AI-proof.
This plugin breaks through it easily.

Ghost fonts are promoted as a real defense — text no AI can read, even a possible CAPTCHA to tell humans from machines. Tested honestly, that claim does not hold. Hiding a message in motion is a textbook computer-vision problem, and the tools that solve it — dense optical flow and phase correlation — are decades old, older than modern AI.

This decoder is a few hundred lines of Python. It read its first ghost-font video on the very first run and returned the exact message. A protection that collapses to a small script is a novelty, not a security boundary — and now your agent reads the hype too.
